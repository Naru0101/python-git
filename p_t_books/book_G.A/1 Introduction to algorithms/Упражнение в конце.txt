Бинарный поиск.
1.1.Для списка из n элементов максимальное количество проверок x можно выразить как log2 n.
 Поэтому для списка из 128 элементов максимальное количество проверок будет log2 128=7
128/2=64, 64/2=32, 32/2=16, 16/2=8, 8/2=4, 4/2=2, 2/2=1.

1.2.Для списка из n элементов максимальное количество проверок x можно выразить как log 2(2n),
что приводит к х = log2 n+1. Таким образом, для списка из 256 элементов максимальное количество проверок будет
log 2 256+1=8+1=9. Также как ти в 1.

О-большое
1.3.Если номера в книге упорядочены по фамилиям и используется бинарный поиск по фамилиям, то время выполнения будет O(log n), где n - количество записей в книге.
Если же используется линейный поиск, то время выполнения будет O(n), так как придется просмотреть каждую запись в книге.

1.4. Время выполнения: O(n), так как придется просмотреть каждую запись в книге, чтобы найти фамилию по номеру.

1.5. Время выполнения: O(n), так как придется прочитать номер каждого человека в книге, а количество операций будет пропорционально количеству записей в книге.

1.6. Время выполнения O(k), где k - количество записей в книге, фамилии которых начинаются с буквы «А».
Это потому, что в данном случае мы будем прочитывать только подмножество записей из книги, а не все записи.
